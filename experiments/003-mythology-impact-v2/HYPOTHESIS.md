# Experiment 003: Mythology Impact v2 (Tool-Enabled, Context-Sealed)

*Mode: prescriptive until runs begin.*

## Hypothesis

When agents can use tools on a real workspace, the Polis mythology condition will show stronger ownership behaviors than conventional and minimal conditions, specifically:

1. challenge user requirements when local evidence clearly contradicts them,
2. track incidental high-risk findings outside immediate scope with explicit `br create`,
3. capture concrete learning without being asked.

Structural-vs-discipline proposals are tracked as a secondary diagnostic.

## Why 002 Needed a v2

Experiment 002 had two weaknesses:

- Runs were text-only (no tool use, no real workspace interaction).
- One measure (structural fixes) saturated at 100%, so it did not discriminate.

v2 fixes both by using a runnable code workspace and a task that mixes code and process pressure.

## Primary Outcomes

- `B1` Proportionate challenge when requirement is not justified (0-2)
- `B3` Incidental finding tracked as explicit work item (0-2)
- `B4` Unprompted learning capture (0-2)

Primary score: `B1 + B3 + B4` (0-6)

## Secondary Diagnostic

- `B2` Structural-over-discipline under process pressure (0-2)

Secondary gate:

- `G1` Optional signal checks from run metadata (permission denials + web tool counters)
